Documentation of Node.js Server for Toys collection

Route Method Info Params/Body
/toys Get Retrieves a list of toys according to the specified query parameters.
  • page (optional): page number to return (default: 1). /toys?page=1
  • limit (optional): number of toys per page (default: 10). /toys?limit=3
  • sort (optional): field to sort by (default: price). /toys?limit=7&sort=price
  • reverse (optional): if set to yes, sorts in descending order. /toys?reverse=yes
  • s (optional): search by name or Info. /toys?s=Huggable
  • category (optional): search by category. /toys?category=abc
Post Adds a new toy. Token is required!
  • Request Body: Toy data to be added in the following format.
  • price: 1-99999
  • img_url is allowed and not required
  • Should look like this: 
                      {
                  "name": "Huggable Puppy",
                  "info": "adorable puppy for play",
                  "category": "Stuffed Animals",
                  "img_url": "any url",
                  "price": 16
                  }
/toys/:id Put Updates an existing toy with the user ID of the toy. Token is required!
  • Request Body: Toy data to be updated in the following format.
  • price: 1-99999
  • img_url is allowed and not required
  • Should look like this: 
                {
              "name": "Updated Toy Name",
              "info": "New toy description",
              "category": "Stuffed Animals",
              "img_url": "new url",
              "price": 20
            }
Delete Deletes an existing toy with the user ID of the toy. Token is required!
  • Request Parameters: Provide the ID of the toy you want to delete in the URL.
  • Example: /toys/123
